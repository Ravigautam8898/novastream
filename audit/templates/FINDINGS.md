## Standard Finding Entry

Every finding MUST contain exactly these fields. No fields may be omitted.

```
### F-XXX — Short Finding Title

| Field | Value |
|-------|-------|
| **Finding ID** | F-XXX |
| **Phase** | [Phase Number and Name] |
| **Category** | [One of: Architecture, Security, Backend, API, Database, Streaming, Frontend, Performance, Accessibility, UX, DevOps, Code Quality, Testing, Documentation] |
| **Severity** | Critical \| High \| Medium \| Low \| Information |
| **Risk** | Critical \| High \| Medium \| Low |
| **Status** | OPEN \| VERIFIED \| APPROVED \| IMPLEMENTING \| IMPLEMENTED \| BUILD PASSED \| WAITING USER TEST \| REGRESSION PASSED \| CERTIFIED \| CLOSED \| REJECTED \| WONT_FIX |
| **Affected Files** | `path/to/file.js:line` (list all affected files) |

**Description:**
Brief description of the issue.

**Root Cause:**
The underlying cause, not just the symptom. Trace through the full call chain if needed.

**Recommended Remediation:**
Exactly what code changes are needed. Include file paths, approach, and trade-offs.

**Implementation Notes:**
*Filled after implementation — summary of what was actually changed.*

| User Approval | Implementation Date | Build Status | Browser Test Status | Regression Status |
|:---:|:---:|:---:|:---:|:---:|
| [Approved / Pending] | YYYY-MM-DD | [PASS / FAIL] | [PASS / FAIL] | [PASS / FAIL] |

**Certification Status:** [CERTIFIED / PENDING]

**Certified By:** [Role — Date]

**Notes:**
*Any additional context, edge cases, or follow-up items.*
```
